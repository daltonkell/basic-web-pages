Images

Unlink all the HTML elements we've encountered this far, image content is defined outside of the web pages
that render it. We refer to these external resources by using any of the above paths.

Images are included in web pages with the <img/> tag and it "src" (source) attribute, which points to the image
you want to display. Notice how it's an "empty" element, like </br>:

<img src="some-photo.png"/>

Image Formats

There are four main image formats used on the web:

  • .jpg: Handle large color palettes without bloating size; good for photos with
    lots of colors, gradients. They do notallow transparent pixels, so that's a bummer.
  • .gif: go-to option for simple animations; limited in terms of color palettes, so never use them
    for photos.
  • .png: great for stuff that isn't a photo and isn't animated (icons, technical diagrams, logos). Bigger than .jpg
  • .svg: "scalable vector graphic"--scalable to any dimension without loss of quality. Phenomenal resource for responsive design.

    • Image Dimesions

    By default, the img element uses the inherited dimensions of its image file. Our JPG, GIF, and PNG images are
    actually 150x150, while our SVG is only 75x75. We'll dicsuss this later, but pixel-based image formats need to be
    twice as big as you want them to appear on retina displays. To get our pixel-based images down to the intended
    size, we can use the "width" attribute of the img tag:





    Setting the "width" attribute sets an explicit dimension for the image. There is also a corresponding "height"
    attribute; setting only one will cause them to scale proportionately, but setting both will cause stretching.
    Dimension values are specified in pixels, and you should never include the unit./

    The "width" and "height" attributes are useful, but it's usually better pratice to set image dimensions in CSS
    so you can alter them with media queries. We'll get to that later.
